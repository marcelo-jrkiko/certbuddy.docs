# CertBuddy Documentation Website - AI Agent Orientation Guide

## Project Overview

**CertBuddy Documentation** is a modern documentation website built with **Vue 3 + TypeScript + Vite**. It features a responsive layout with a lateral navigation menu powered by static JSON configuration.

### Key Technologies
- **Frontend Framework**: Vue 3 (Composition API)
- **Language**: TypeScript
- **Build Tool**: Vite
- **Styling**: Vue SFC scoped CSS
- **Node Version**: 14.0+

## Core Concepts

### 1. Menu Configuration System (CRITICAL)
The **entire navigation structure** is defined in a static JSON file:
- **File**: `src/config/menu.json`
- **Purpose**: Defines sections, items, and component mappings
- **Structure**:
  ```json
  {
    "sections": [
      {
        "id": "section-id",
        "title": "Section Title",
        "items": [
          {
            "id": "item-id",
            "title": "Item Title",
            "component": "ComponentName"  // maps to src/pages/ComponentName.vue
          }
        ]
      }
    ]
  }
  ```

### 2. Component-Based Documentation Pages
- Each documentation page is a **separate Vue component**
- Located in `src/pages/` directory
- Named as `[PageName]Page.vue` (e.g., `IntroductionPage.vue`)
- Components are **dynamically loaded** when menu items are clicked
- Use async component loading with error handling

### 3. Dynamic Component Loading
Inside `MainLayout.vue`, components are loaded using:
```typescript
const loadComponent = async (componentName: string) => {
  currentComponent.value = defineAsyncComponent(
    () => import(`@/pages/${componentName}.vue`)
  );
};
```

### 4. TypeScript Types
Menu structure types are defined in `src/types/menu.ts`:
- `MenuItem` - Individual menu items
- `MenuSection` - Sections containing items
- `MenuConfig` - Complete menu configuration

## Development Guidelines

### Adding a New Documentation Page

**Step 1**: Create component in `src/pages/`
```vue
<!-- src/pages/MyNewPage.vue -->
<template>
  <article class="documentation-page">
    <h2>Page Title</h2>
    <p>Content here...</p>
  </article>
</template>

<script setup lang="ts">
</script>

<style scoped>
.documentation-page {
  line-height: 1.6;
  color: #2d3748;
}
</style>
```

**Step 2**: Add to `src/config/menu.json`
```json
{
  "id": "my-new-page",
  "title": "My New Page",
  "component": "MyNewPage"  // Must match file name without .vue
}
```

### Styling Conventions

1. **Use scoped styles** - All components use `<style scoped>`
2. **Color Scheme**:
   - Primary Gradient: `#667eea` → `#764ba2`
   - Dark Text: `#2d3748`
   - Light Background: `#f7fafc`
   - Light Border: `#e2e8f0`
3. **Responsive Breakpoints**:
   - Desktop: Default
   - Tablet: `@media (max-width: 768px)`
   - Mobile: `@media (max-width: 640px)`

### Component Props & Emits

**SideMenu Component**:
- Props: `sections`, `activeItemId`
- Emits: `@select-item`

**Header Component**:
- Props: `appTitle`, `version`, `logoSrc`

**Footer Component**:
- Props: `copyrightText`

### TypeScript Best Practices

1. Always use **type definitions** for props and emits
2. Use `withDefaults()` for prop defaults
3. Use proper TypeScript interfaces for data structures
4. Avoid `any` type - use specific types

Example:
```typescript
interface Props {
  title: string;
  items: MenuItem[];
}

interface Emits {
  (e: 'select', item: MenuItem): void;
}

const props = withDefaults(defineProps<Props>(), {
  // defaults here
});

const emit = defineEmits<Emits>();
```

## Common Tasks for AI Agents

### Task: Fix TypeScript Errors
- Check `tsconfig.json` for strict mode settings
- Ensure all props/emits use proper TypeScript interfaces
- No duplicate definitions (like duplicate `defineEmits`)

### Task: Add New Menu Section
1. Modify `src/config/menu.json` - add new section object
2. Create new page components in `src/pages/`
3. Verify component names match JSON configuration

### Task: Update Styling
- Maintain responsive design with media queries
- Keep color scheme consistent
- Ensure accessibility (contrast, readable fonts)

### Task: Component Refactoring
- Maintain Vue 3 Composition API patterns
- Keep components focused and reusable
- Use proper TypeScript typing

## Troubleshooting Guide

| Issue | Solution |
|-------|----------|
| Component not loading | Check menu.json - component name must match file name |
| Component not rendering | Verify component exists in src/pages/ |
| Type errors | Ensure TypeScript interfaces defined for props/emits |
| Menu not updating | Restart dev server after menu.json changes |
| Styling issues | Check responsive breakpoints - may need adjustment |

## Performance Considerations

1. **Async Component Loading**: Components load on demand - improves initial load
2. **Scoped Styles**: No style conflicts between components
3. **Vue 3 Composition API**: Better tree-shaking and bundle optimization
4. **Vite HMR**: Fast hot module replacement during development

## Security Considerations

- No sensitive data in menu.json
- User input sanitized (if any user-generated content added)
- XSS protection through Vue's built-in sanitization
- CSP headers should be configured on hosting
